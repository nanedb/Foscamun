DATABASE MIGRATION REQUIRED
===========================

A seguito delle modifiche al codice, è necessario eseguire una migrazione del database esistente.

COSA È CAMBIATO:
----------------
1. ICJ non viene più salvato nella tabella "Committees"
2. ICJ appare nella lista dei comitati solo se la tabella ICJ contiene dati
3. ICJ viene identificato con CommID = -1 nell'applicazione

COME ESEGUIRE LA MIGRAZIONE:
-----------------------------

OPZIONE 1 - Script SQL Automatico (Consigliato):
1. Aprire il database con un client SQLite (es. DB Browser for SQLite)
   Percorso: C:\Users\Giovanni\AppData\Local\Foscamun2026\Foscamun.db
2. Aprire ed eseguire il file: DatabaseMigration_RemoveICJFromCommittees.sql
3. Verificare che ICJ sia stato rimosso dalla tabella Committees

OPZIONE 2 - Comandi SQL Manuali:
Eseguire questi comandi SQL sul database:

    DELETE FROM Committees WHERE Name = 'ICJ';
    DELETE FROM CountryLists WHERE CommID NOT IN (SELECT CommID FROM Committees);

OPZIONE 3 - Nessuna Azione (per database nuovi):
Se il database non ha ancora una riga ICJ nella tabella Committees, non è necessaria alcuna azione.

VERIFICA POST-MIGRAZIONE:
-------------------------
1. Avviare l'applicazione
2. Verificare che:
   - Se la tabella ICJ contiene dati, ICJ appare nella lista dei comitati
   - Se la tabella ICJ è vuota, ICJ NON appare nella lista
   - Il pulsante "Edit Committee" è disabilitato quando ICJ è selezionato
   - Il pulsante "ICJ" è SEMPRE abilitato (per creare o modificare ICJ)
   - Cliccando "ICJ": si apre EditICJPage con i dati esistenti o vuota per crearne uno nuovo
   - Il pulsante "Remove" funziona correttamente per ICJ (svuota la tabella ICJ)

NOTE:
-----
- È consigliato fare un backup del database prima della migrazione
- La migrazione è compatibile con l'esecuzione multipla (può essere eseguita più volte)
